# 游泳教练辅助平台 - 启动指南

## 📋 前置要求

1. **Java 17** 或更高版本
2. **Maven 3.6+**
3. **MySQL 5.7+** 或 **MySQL 8.0+**

## 🗄️ 数据库配置

### 1. 创建数据库

在 MySQL 中执行以下 SQL 脚本：

```bash
mysql -u root -p < database.sql
```

或者手动执行 `database.sql` 文件中的内容。

该脚本会：
- 创建数据库 `swimming_coach_platform`
- 创建所有必需的表结构
- 插入测试数据（教练和学员账号）

### 2. 修改数据库连接配置

编辑 `src/main/resources/application.properties` 文件：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/swimming_coach_platform?useSSL=false&serverTimezone=UTC&characterEncoding=utf8
spring.datasource.username=root          # 修改为您的MySQL用户名
spring.datasource.password=your_password # 修改为您的MySQL密码
```

## 🚀 启动应用

### 方法一：使用 IDEA（推荐）

1. 用 IntelliJ IDEA 打开项目
2. 等待 Maven 依赖下载完成
3. 点击右上角的运行配置 "swimming"
4. 点击绿色运行按钮 ▶️ 或按 `Shift + F10`

### 方法二：使用 Maven 命令

在项目根目录打开终端，执行：

```bash
mvn spring-boot:run
```

## 🌐 访问应用

启动成功后，在浏览器中访问：

```
http://localhost:8081/mySpringBoot
```

**注意：** 
- 端口号：8081（在 application.properties 中配置）
- 上下文路径：/mySpringBoot

## 🔧 常见问题

### 1. 端口被占用

如果 8081 端口被占用，修改 `application.properties`：

```properties
server.port=8080  # 改为其他可用端口
```

### 2. 数据库连接失败

检查：
- MySQL 服务是否启动
- 数据库名、用户名、密码是否正确
- 防火墙是否阻止连接

### 3. Maven 依赖下载慢

配置国内镜像源，编辑 `~/.m2/settings.xml`：

```xml
<mirrors>
    <mirror>
        <id>aliyun</id>
        <mirrorOf>central</mirrorOf>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```
