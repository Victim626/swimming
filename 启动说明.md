# 游泳教练辅助平台 - 本地部署指南

## 前置要求

| 环境 | 版本 | 说明 |
|------|------|------|
| Java | 17+ | 项目使用 JDK 17 |
| Maven | 3.6+ | 也可用项目自带的 mvnw |
| Node.js | 18+ | 前端 Vite 构建需要 |
| MySQL | 8.0+ | 数据库 |

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/Victim626/swimming.git
cd swimming

# 如需代理（国内网络环境）：
git config http.proxy http://127.0.0.1:7897
git config https.proxy http://127.0.0.1:7897
```

### 2. 导入数据库

项目根目录的 `swimming_coach_platform_schema.sql` 包含完整的表结构和测试数据。

```bash
# 创建并导入
mysql -u root -p < swimming_coach_platform_schema.sql
```

该文件会自动创建 `swimming_coach_platform` 数据库，包含 18 张表和完整的测试数据。

### 3. 配置后端连接

编辑 `src/main/resources/application.properties`，修改数据库密码：

```properties
spring.datasource.username=root
spring.datasource.password=your_password
```

### 4. 启动后端

```bash
# 设置 JAVA_HOME（mvnw 需要）
export JAVA_HOME="D:/IT_AI_learn/jdk-17.0.9"

# 如需代理：
MAVEN_OPTS="-Dhttp.proxyHost=127.0.0.1 -Dhttp.proxyPort=7897 -Dhttps.proxyHost=127.0.0.1 -Dhttps.proxyPort=7897"

# 启动
./mvnw spring-boot:run
```

启动后后端运行在 `http://localhost:8081/mySpringBoot`。

### 5. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端运行在 `http://localhost:80`，已配置 Vite 代理将 `/mySpringBoot` 请求转发到后端。

## 常见问题

### 1. 端口被占用

```bash
# 查看端口占用
netstat -ano | findstr :8081
netstat -ano | findstr :80

# 杀掉进程
taskkill //F //PID 进程ID
```

### 2. JAVA_HOME 未设置

mvnw 需要 `JAVA_HOME` 环境变量，在 Git Bash 中执行：

```bash
export JAVA_HOME="你的JDK安装路径"
```

### 3. Maven 依赖下载慢

配置阿里云镜像源，编辑 `~/.m2/settings.xml`：

```xml
<mirrors>
    <mirror>
        <id>aliyun</id>
        <mirrorOf>central</mirrorOf>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

或通过代理下载（在 MAVEN_OPTS 中配置代理）。

### 4. 数据库连接失败

- 确认 MySQL 服务已启动
- 检查 application.properties 中的用户名、密码、数据库名
- 确认 SQL 文件已正确导入

### 5. 前后端联不通

前端 Vite 配置了代理（`frontend/vite.config.ts`），确保：
- 后端已启动且 8081 端口可访问
- 前端请求路径以 `/mySpringBoot` 开头

### 6. 终端中文乱码

Windows Git Bash 下中文显示乱码是 GBK/UTF-8 编码问题，不影响实际数据存储。
