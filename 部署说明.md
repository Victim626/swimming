# 游泳训练平台部署说明

## 一、部署目标

本文档用于在 Windows 或 Linux 服务器上部署当前项目：
- 后端：Spring Boot + MyBatis
- 前端：Vue 3 + Vite
- 数据库：MySQL

项目当前后端默认配置：
- 端口：`8081`
- 上下文路径：`/mySpringBoot`
- 数据库名：`swimming_coach_platform`

---

## 二、部署前准备

### 1. 环境要求

- JDK 17
- Maven 3.8+ 或项目自带 Maven 环境
- Node.js 18+
- MySQL 8.x
- Nginx（前端生产环境建议使用）

### 2. 数据库准备

先在 MySQL 中创建数据库：
```sql
CREATE DATABASE swimming_coach_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后导入项目对应的建表 SQL。当前仓库中与业务相关的表结构可参考：
- `posts`
- `post_comments`
- `users`
- `plan_details`

如果数据库用户名、密码或地址不同，需要同步修改：
`src/main/resources/application.properties`

当前配置示例：
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/swimming_coach_platform?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

## 三、后端部署

### 1. 修改数据库配置

检查并确认以下配置项正确：
- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`

后端默认启动端口和路径：
```properties
server.port=8081
server.servlet.context-path=/mySpringBoot
```

### 2. 打包后端

在项目根目录执行：
```bash
mvn clean package -DskipTests
```

打包完成后会生成后端可执行 Jar 包，通常在：
```text
target/*.jar
```

### 3. 启动后端

在服务器上执行：
```bash
java -jar target/swimming-0.0.1-SNAPSHOT.jar
```

如果要让后端在后台运行，可以使用：
```bash
nohup java -jar target/swimming-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

### 4. 验证后端

浏览器或接口工具访问：
```text
http://localhost:8081/mySpringBoot
```

如果有接口可直接访问，也可以用具体接口验证服务是否正常。

---

## 四、前端部署

### 1. 安装依赖

进入前端目录：
```bash
cd frontend
npm install
```

### 2. 修改前端接口地址

如果前端项目中配置了 API 基础地址，需要把它改成后端实际地址：
```text
http://服务器IP:8081/mySpringBoot
```

如果前后端同域部署，也可以通过 Nginx 反向代理到后端。

### 3. 打包前端

在 `frontend` 目录执行：
```bash
npm run build
```

打包产物默认生成在：
```text
frontend/dist
```

### 4. 部署前端静态文件

将 `dist` 目录内容复制到 Nginx 的站点目录，例如：
```text
/usr/share/nginx/html/swimming/
```

Windows 环境下也可以直接使用其他静态文件服务方式，但生产环境建议使用 Nginx。

---

## 五、Nginx 配置示例

以下示例将前端静态页面和后端接口一起代理：

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root /usr/share/nginx/html/swimming;
        try_files $uri $uri/ /index.html;
    }

    location /mySpringBoot/ {
        proxy_pass http://127.0.0.1:8081/mySpringBoot/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果前端和后端部署在不同域名或端口，需要确认后端跨域策略是否允许前端访问。

## 六、启动顺序建议

建议按以下顺序启动：
1. MySQL
2. 后端 Spring Boot
3. 前端静态服务或 Nginx

---

## 七、常见检查点

### 1. 数据库连不上
- 检查 MySQL 是否启动
- 检查数据库名是否为 `swimming_coach_platform`
- 检查用户名和密码是否正确
- 检查 JDBC URL 是否可访问

### 2. 后端启动失败
- 检查 Java 是否为 17
- 检查端口 `8081` 是否被占用
- 检查数据库表是否已创建

### 3. 前端页面空白或接口报错
- 检查前端 API 地址是否配置正确
- 检查后端是否已启动
- 检查 Nginx 反向代理路径是否与后端上下文路径一致

---
